<task type="auto" tdd="true">
  <name>Task 1: Watchlist router — GET, POST, DELETE</name>
  <files>backend/app/routers/watchlist.py</files>
  <behavior>
    - GET /api/watchlist: returns {"watchlist": [{"ticker": str, "price": float | None}, ...]} for user "default"
    - POST /api/watchlist: body {ticker: str}, inserts OR IGNORES into watchlist, calls source.add_ticker, returns 201 {"ticker": str}
    - DELETE /api/watchlist/{ticker}: removes ticker from DB, calls source.remove_ticker, returns {"ticker": str}
    - DELETE non-existent ticker: returns 404 with detail
    - POST duplicate ticker: idempotent (INSERT OR IGNORE), still calls add_ticker, returns 201
    - Ticker values normalized to .upper() on all writes and lookups
  </behavior>
  <action>
Create `backend/app/routers/watchlist.py` following project conventions.

Structure:
1. Module docstring
2. Imports: logging, uuid, datetime/timezone, fastapi (APIRouter, HTTPException, Request), pydantic (BaseModel), app.db (get_connection), app.market (PriceCache, MarketDataSource)
3. `logger = logging.getLogger(__name__)`
4. `router = APIRouter(tags=["watchlist"])`
5. Pydantic model: `class WatchlistAddRequest(BaseModel): ticker: str`
6. `GET /watchlist` — reads all tickers for user "default", fetches price from `request.app.state.price_cache.get_price(ticker)` (None is valid if not cached yet), returns list
7. `POST /watchlist` (status_code=201) — INSERT OR IGNORE, commit, then `await request.app.state.source.add_ticker(ticker.upper())`
8. `DELETE /watchlist/{ticker}` — check if ticker exists first; if not, raise HTTPException(404, detail=f"Ticker {ticker} not in watchlist"); DELETE row, commit, then `await request.app.state.source.remove_ticker(ticker.upper())`

Critical patterns (per RESEARCH.md pitfalls):
- Access `source` via `request.app.state.source` — never import from main.py (circular)
- Use `INSERT OR IGNORE` for idempotent adds (avoids 500 on UNIQUE constraint)
- Check existence before DELETE to return 404 instead of silently doing nothing
- Always `.upper()` on ticker input
- `get_connection()` reads DB_PATH lazily — test-safe
