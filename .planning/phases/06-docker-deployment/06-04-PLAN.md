  <files></files>
  <action>
This tests DOCK-03: the named volume `finally-data` preserves data across container restarts.

Step 1 — Record current state:
```bash

```
Note the cash balance and positions from the output.

Step 2 — Stop the container (WITHOUT removing the volume):
```bash

```
Confirm output says "Container stopped. Volume (data) preserved."

```bash
bash /Users/steve/projects/finally/scripts/start_mac.sh
```

Step 4 — Wait ~5 seconds for the app to start, then check portfolio:
```bash
sleep 5 && curl -sf http://localhost:8000/api/portfolio
```

Verify:
- Cash balance matches what was recorded in Step 1 (the trade from Task 2 is preserved)
- Positions from Step 1 are still present
- The app is fully functional again

If portfolio data is NOT preserved, the volume mount is broken. Check `docker inspect finally` for the volume mount details.
  </action>
  <verify>curl -sf http://localhost:8000/api/portfolio | python3 -c "import sys,json; d=json.load(sys.stdin); assert d.get('cash_balance',10000) < 10000, 'cash not changed — trade not persisted'; print('PERSISTENCE OK: cash =', d['cash_balance'])"</verify>
