# Phase 4 — UI Design Contract

> Visual and interaction contract for the frontend shell and watchlist phase. Generated by gsd-ui-researcher.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — custom dark theme via Tailwind v4 CSS custom properties |
| Preset | not applicable |
| Component library | none — hand-rolled components, no Radix/shadcn |
| Icon library | none — status indicators use CSS-only colored dots |
| Font | system-ui (default browser stack) |

**Rationale:** The Bloomberg terminal aesthetic is a custom dark design. shadcn presets produce light-mode-default components that conflict with the `#0d1117` base. Tailwind v4 `@theme` block drives all tokens; RESEARCH.md defines these precisely. Confirmed from: RESEARCH.md § Tailwind CSS v4 Setup, PLAN.md § Visual Design.

## Typography

All sizes use the system font stack. Source: RESEARCH.md color token `text-primary` (#e6edf3) and `text-muted` (#8b949e). Trading terminal convention: dense, small text — readable data-first.

| Role | Size | Weight | Line Height | Color Token | Usage |
|------|------|--------|-------------|-------------|-------|
| Body | 14px | 400 | 1.4 | text-primary (#e6edf3) | Watchlist prices, portfolio values, change % |
| Label | 12px | 400 | 1.3 | text-muted (#8b949e) | Column headers, ticker sub-labels, timestamps |
| Ticker | 14px | 600 | 1.3 | text-primary (#e6edf3) | Ticker symbols (e.g. AAPL) in watchlist rows |
| Display | 18px | 600 | 1.2 | text-primary (#e6edf3) | Header portfolio total value |

**Rule:** Only 2 weights: 400 (regular) and 600 (semibold). No bold (700). No italic. No underline.

## Interaction Contract

### Price Flash Animation

- Trigger: SSE `direction` field changes to `'up'` or `'down'` on a new price event
- Duration: 500ms fade from colored background to transparent
- Up: keyframe from `rgba(34, 197, 94, 0.4)` → `transparent` (uptick green at 40% opacity)
- Down: keyframe from `rgba(239, 68, 68, 0.4)` → `transparent` (downtick red at 40% opacity)
- Applies to: price cell in WatchlistRow only
- Implementation: CSS `@keyframes flash-up` and `@keyframes flash-down` defined in `globals.css`; React toggles class name; `useEffect` cleanup clears pending `setTimeout` before setting new one (prevents stale class stacking)
- Source: RESEARCH.md § Price Flash Animation

### Ticker Selection

- Trigger: click anywhere on WatchlistRow
- Effect: selected row gets 4px left border in `#ecad0a` (accent) and `#161b22` background (secondary)
- State: `selectedTicker: string | null` held in `page.tsx`, passed down as props
- Deselect: clicking the already-selected row has no effect (selection persists until another row is clicked)
- MainArea shows: `"{ticker} selected — chart coming in Phase 5"` placeholder text in `#8b949e` (text-muted)

### Connection Status Dot

- Green (#22c55e): EventSource `readyState === 1` (OPEN) — `onopen` fired
- Yellow (#ecad0a): EventSource reconnecting — `onerror` fired but not yet OPEN
- Red (#ef4444): EventSource closed/failed — not connected

### SSE Reconnection

- Native EventSource auto-reconnects using the `retry: 1000` value sent by the backend
- No manual reconnect logic needed in the frontend
- Status dot transitions yellow → green when reconnection succeeds

### Sparkline Update

- Initial: `series.setData(allHistory)` on mount
- Live: `series.update({ time: UTCTimestamp, value: price })` per SSE tick — never `setData` on update
- History: accumulated in `useRef<Map<string, number[]>>` inside `usePriceStream` hook
- Container: 80px × 32px, transparent background, no axes, no crosshair, no scroll/scale
