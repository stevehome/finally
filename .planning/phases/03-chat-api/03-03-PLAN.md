**Create `backend/app/routers/chat.py`** following RESEARCH.md Pattern 8:

```python
"""Chat router: POST /api/chat — LLM conversation with auto-execute."""
```

Components:

**ChatRequest(BaseModel)**: `message: str`

**_save_message(user_id, role, content, actions=None)**: Helper to INSERT into chat_messages table. Uses `uuid.uuid4()` for id, `datetime.now(timezone.utc).isoformat()` for created_at.

**POST /api/chat handler** (`async def chat(request: Request, body: ChatRequest) -> dict`):

1. Extract `price_cache` and `source` from `request.app.state`
2. Save the user message: `_save_message(user_id, "user", body.message)`
3. Build portfolio context: `build_portfolio_context(price_cache, user_id)`
4. Build system prompt: `build_system_prompt(portfolio_context)`
5. Load conversation history: `load_history(user_id)`
6. Construct messages list: `[{"role": "system", "content": system_prompt}] + history`
   Note: Do NOT append the current user message again — it's already in history (saved in step 2 and loaded in step 5). If load_history returns empty (first message), append `{"role": "user", "content": body.message}` manually.

   Actually, simpler approach: save user message AFTER loading history, so history doesn't include the current message yet. Then append the current user message to the messages list:

   Corrected flow:
   1. Load history first (before saving current message)
   2. Save user message to DB
   3. Build messages: `[system] + history + [{"role": "user", "content": body.message}]`
   4. Call LLM

7. Call LLM in a thread: `llm_response = await asyncio.to_thread(call_llm, messages)` — this prevents blocking the event loop (RESEARCH.md Pitfall 2)
8. Auto-execute trades: Loop over `llm_response.trades`, call `execute_trade_internal(price_cache, t.ticker.upper(), t.quantity, t.side.lower(), user_id)` for each, collect results
9. Apply watchlist changes: Loop over `llm_response.watchlist_changes`, call `add_ticker_internal(source, c.ticker.upper(), user_id)` if action is "add", or `remove_ticker_internal(source, c.ticker.upper(), user_id)` if action is "remove". Collect `{"ticker", "action", "applied": bool}` results.
10. Build actions payload:
    ```python
    actions_payload = {
        "trades_executed": [r for r in trade_results if r["error"] is None],
        "trades_failed": [r for r in trade_results if r["error"] is not None],
        "watchlist_changes": watchlist_results,
    }
    ```
11. Save assistant message: `_save_message(user_id, "assistant", llm_response.message, actions=json.dumps(actions_payload))`
12. Return `{"message": llm_response.message, "actions": actions_payload}`

Use `router = APIRouter(tags=["chat"])`.
