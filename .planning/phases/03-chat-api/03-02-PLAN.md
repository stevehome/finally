**portfolio.py — extract `execute_trade_internal`:**

Add a new function `execute_trade_internal` ABOVE the existing `execute_trade` handler. It contains the core trade logic extracted from `execute_trade`, but returns a result dict instead of raising HTTPException.

```python
def execute_trade_internal(
    price_cache: PriceCache,
    ticker: str,
    quantity: float,
    side: str,
    user_id: str = "default",
) -> dict:
    """Execute a trade, returning a result dict with error=None on success.

    Does NOT raise HTTPException — caller decides how to handle errors.
    Returns: {"ticker", "side", "quantity", "price", "error"}
    """
```

Logic is the same as `execute_trade` lines 121-249, but:
- Instead of `raise HTTPException(400, "Insufficient cash")` → return `{"ticker": ticker, "side": side, "quantity": quantity, "price": current_price, "error": "Insufficient cash"}`
- Instead of `raise HTTPException(400, "Insufficient shares")` → return with `"error": "Insufficient shares"`
- Instead of `raise HTTPException(400, "Invalid side...")` → return with `"error": "Invalid side"`
- Instead of `raise HTTPException(500, "User profile not found")` → return with `"error": "User profile not found"`
- On sqlite3.Error, log and return with `"error": "Database error"`
- On success, return with `"error": None` and include the trade price
- Still calls `record_snapshot` after successful trade (same pattern as current handler)

Then refactor the existing `execute_trade` HTTP handler to call `execute_trade_internal` and convert error results to HTTPException:
```python
@router.post("/portfolio/trade")
async def execute_trade(request: Request, trade: TradeRequest) -> dict:
    price_cache: PriceCache = request.app.state.price_cache
    result = execute_trade_internal(
        price_cache, trade.ticker.upper(), trade.quantity, trade.side.lower()
    )
    if result["error"]:
        status = 500 if result["error"] == "Database error" else 400
        raise HTTPException(status_code=status, detail=result["error"])
    return {"status": "ok", **{k: v for k, v in result.items() if k != "error"}}
```

This preserves the exact same HTTP behavior for existing tests.
