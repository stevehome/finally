Tests to create (each with a docstring stating the requirement ID):

1. **test_chat_returns_message** (CHAT-01): POST `/api/chat` with `{"message": "Hello"}` returns 200 with a `message` field in the response JSON. Use `TestClient(app)` with `LLM_MOCK=true` monkeypatched.

2. **test_chat_includes_portfolio_context** (CHAT-02): After buying 1 share of AAPL, POST `/api/chat` with `{"message": "What is my portfolio?"}`. The test asserts that the LLM was called with messages containing portfolio context. To verify this, monkeypatch `app.llm.call_llm` to capture the messages list it receives, then assert the system message contains "AAPL" and "cash" (case-insensitive).

3. **test_chat_auto_executes_trade** (CHAT-03): Monkeypatch `app.llm.call_llm` to return a `ChatResponse` with `trades=[TradeAction(ticker="AAPL", side="buy", quantity=1)]`. After the chat call, GET `/api/portfolio` and assert a position in AAPL exists.

4. **test_chat_applies_watchlist_changes** (CHAT-04): Monkeypatch `call_llm` to return a `ChatResponse` with `watchlist_changes=[WatchlistAction(ticker="PYPL", action="add")]`. After the chat call, GET `/api/watchlist` and assert PYPL is in the list.

5. **test_chat_history_persisted** (CHAT-05): Send two POST `/api/chat` messages sequentially. On the second call, monkeypatch `call_llm` to capture messages. Assert that the captured messages list contains the first user message and first assistant response (i.e., history is included).

6. **test_chat_mock_mode** (CHAT-06): With `LLM_MOCK=true` monkeypatched, POST `/api/chat` with `{"message": "Hello"}`. Assert response is 200 and `message` field is a non-empty string. Assert that litellm.completion was NOT called (monkeypatch it to raise if called).

7. **test_chat_failed_trade_in_response** (CHAT-07): Monkeypatch `call_llm` to return a `ChatResponse` with `trades=[TradeAction(ticker="AAPL", side="sell", quantity=999)]` (user has no AAPL shares). Assert that response is 200 (not 400), and `actions.trades_failed` contains an entry with an error string.

For tests 2-5, set `LLM_MOCK=true` as the default and override `call_llm` via monkeypatch where needed to control its return value.
